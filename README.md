# DiscoveryAdvisor

> Quel artefact discovery utiliser maintenant ?

---

## Le problème

Un OST quand il faut un Story Map. Un Story Map quand il faut un JTBD. La plupart des PMs connaissent les artefacts discovery — mais choisissent le mauvais au mauvais moment.

**Le résultat :** des workshops qui n'avancent pas, des équipes qui s'alignent sur le mauvais niveau, et du temps de discovery gaspillé sur le mauvais outil.

## Ce que ça fait

DiscoveryAdvisor prend ton contexte et recommande l'artefact discovery le plus adapté à ta situation.

4 questions. 1 recommandation. Les étapes pour démarrer.

```
Objectif → Clarté sur le problème → Audience → Horizon → Artefact recommandé
```

## Les 6 artefacts couverts

| Artefact | Auteur | Quand l'utiliser |
|----------|--------|-----------------|
| Opportunity Solution Tree | Teresa Torres | Explorer opportunités et solutions autour d'un outcome |
| Story Mapping | Jeff Patton | Planifier le delivery d'une solution connue |
| Impact Mapping | Gojko Adzic | Aligner stakeholders business sur les priorités |
| How Might We + Problem Statement | IDEO | Partir de zéro, cadrer le problème |
| Jobs To Be Done Map | Christensen / Ulwick | Comprendre les motivations profondes des utilisateurs |
| Assumption Mapping | Bland & Osterwalder | Identifier et prioriser les hypothèses à tester |

## La logique de décision

La recommandation repose sur 4 dimensions combinées :

**Objectif** — comprendre le problème, explorer des solutions, aligner, planifier le delivery.

**Clarté** — je pars de zéro, j'ai des hypothèses, le problème est compris, problème et solution sont clairs.

**Audience** — solo, équipe produit, stakeholders, tout le monde.

**Horizon** — feature (semaines), produit (mois), vision (long terme).

## Stack

HTML / CSS / JS pur. Aucune dépendance, aucune étape de build.

Ouvre `index.html` dans un navigateur, ça tourne.
